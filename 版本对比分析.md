# CROPGRO-Strawberry 版本对比分析报告

## 一、版本概述

| 版本 | 文件路径 | 运行方式 |
|------|----------|----------|
| **原版** | `cropgro-strawberry-original.py` | 通过 Docker 运行 |
| **修改版** | `cropgro-strawberry-modified.py` | 直接在 Python 中运行 |

## 二、核心区别对比

### 2.1 运行环境

> **原版**：指 `cropgro-strawberry-original.py`，需要通过 Docker 容器运行的代码  
> **修改版**：指 `cropgro-strawberry-modified.py`，可以直接在本地 Python 环境运行的代码

| 项目 | 原版（Docker代码） | 修改版（Python代码） |
|------|-------------------|---------------------|
| **依赖环境** | 需要 Docker 容器 | 仅需 Python 3.8+ |
| **操作系统** | Linux (Docker 容器内) | Windows/macOS/Linux 均可 |
| **Matplotlib 后端** | 默认配置（可能无 GUI 窗口） | 指定 TkAgg（确保图表窗口弹出） |
| **代码执行位置** | Docker 虚拟容器中 | 本地物理机器上 |

### 2.2 数据来源

> **原版**：使用代码内硬编码的合成模拟数据进行演示  
> **修改版**：读取真实的 DSSAT 实验数据文件进行模拟

| 项目 | 原版（Docker代码） | 修改版（Python代码） |
|------|-------------------|---------------------|
| **数据类型** | 合成模拟数据（随机生成） | 真实 DSSAT 实验数据 |
| **天气数据** | 在代码中使用 NumPy 随机生成 | 从气象站文件 (.WTH) 读取真实气象数据 |
| **土壤数据** | 使用代码内硬编码的默认参数 | 从土壤参数文件 (.SRT) 读取真实土壤数据 |
| **实验配置** | 种植日期、品种等参数硬编码在代码中 | 从实验配置文件 (.SRX) 自动解析读取 |

### 2.3 代码结构

| 项目 | 原版 | 修改版 |
|------|------|--------|
| **主函数** | `run_example_simulation()` | `run_with_dssat_data()` |
| **参数配置** | 代码内硬编码 | 从外部文件读取 |
| **实验数量** | 单个模拟 | 批量处理多个实验 |
| **错误处理** | 无 | try-except 包裹 |

### 2.4 输出内容

| 项目 | 原版 | 修改版 |
|------|------|--------|
| **图表数量** | 1 张 | 多张 (完整图+对比图+单项图) |
| **CSV 输出** | 无 | 每个实验单独保存 |
| **中文标签** | 英文 | 中文 |
| **自动打开** | 否 | 是 |
| **进度显示** | 无 | 详细日志输出 |

### 2.5 功能特性

| 项目 | 原版 | 修改版 |
|------|------|--------|
| **年份对比** | 无 | ✅ 支持 |
| **动态演示** | 无 | ✅ 支持 |
| **单独指标图** | 无 | ✅ 支持 |
| **图表保存** | 手动保存 | 自动保存到指定目录 |

## 六、总结

### 6.1 修改版的优势

1. **无需 Docker**: 直接在本地 Python 环境运行，降低入门门槛
2. **真实数据**: 使用 DSSAT 官方实验数据，结果更可靠
3. **丰富输出**: 自动生成多张图表和 CSV 数据文件
4. **中文界面**: 图表标签和日志均为中文，更易理解
5. **批量处理**: 支持同时处理多个年份的实验数据

### 6.2 原版的优势

1. **官方标准**: 与 DSSAT Fortran 版本完全一致
2. **环境隔离**: Docker 保证环境一致性
3. **代码纯净**: 无额外修改，适合学术引用

### 6.3 推荐使用方式

| 用户类型 | 推荐版本 |
|----------|----------|
| **初学者/学生** | 修改版 (易于运行) |
| **研究者** | 原版 (保证准确性) |
| **开发者** | 修改版 (便于调试) |
| **演示展示** | 修改版 (丰富输出) |
